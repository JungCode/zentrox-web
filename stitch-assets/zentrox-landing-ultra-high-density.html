Frequently Asked Questions

Technical details for the discerning architect.

Is there a hard limit on execution calls?

We don't charge per individual call, but rather based on 'Logic Units' which account for complexity. Our Professional plan handles up to 1 million standard executions before any overage applies.

How does Zentrox handle API rate limiting for third-party services?

Our orchestrator features built-in smart queuing and exponential backoff. It automatically detects 429 status codes and throttles execution at the node level to ensure your external API keys remain in good standing.

Can I export my automation logic to code?

Yes. Every workflow can be exported as a standard YAML configuration file or a set of TypeScript functions, allowing you to include your automation logic in your version control system (Git).

What AI models power the decision nodes?

We use a multi-model approach. For simple routing, we use proprietary lightweight BERT-based models for speed. For complex reasoning, you can select between GPT-4o, Claude 3.5, or your own fine-tuned models hosted on-premise.

How is data security handled during AI processing?

Data is never used for training global models. We utilize Zero-Retention APIs for cloud models and offer local execution nodes where data never leaves your infrastructure.

Is there support for custom protocols?

Zentrox supports HTTP/2, gRPC, WebSockets, and MQTT out of the box. For legacy or proprietary protocols, you can develop custom adapters using our WASM SDK.
